Project overview

For the final project, I will be attempting to create a functional tablet. I will be using a Raspberry Pi as the mainboard and adding on a touchscreen, battery and GPS reciever. I will also be 3D Printing a case for the tablet in order to protect it. I will be trying to run Android 9.0 on the machine through RaspAnd Pie 9. However, as Android is a very demanding Linux OS and the RPi does have severe limitations in pure computing performance (I will be using the Model 3B) performance issues may force me to seek alternatives such as Ubuntu MATE or Mint (Debian Edition).


    The end product should be able to:
  • Be used with a touch interface only
  • Connect to the internet and run webapps (except for video-streaming) properly
  • Run with satisfactory performance (no sudden frame drops, throttling, consistent stuttering)
  • Multitask

Equipment list

  • Raspberry Pi (Model 3B)
  • Raspberry Pi official 7" touchscreen
  • Power Supply/Battery board with USB outputs
  • 3800mAh Li-ion battery
  • NEO-6M GPS reciever+GPS antenna
  • 16GB microSD card

Comprimises and limitations

Exposed PCB and wiring

Although a case that would cover both the display subassembly and PCB was originally intended, limitations with 3D printing meant that the bulge in the middle of the case that would be required to fit the PCB subassembly would not have printed properly. Printing a case for the display only was possible and would've allowed at least some protection for the panel. The PCB, however, remains exposed so the end product is still extremely fragile. This also means that the majority of cables are exposed and can be easily disconnected inadverntently.

Raspian

Due to limitations in the code for RaspAnd Pie, the RPi would not send display outputs through the display port to the official touchscreen and would only send signals through the HDMI port. I tried to boot the RPi with Android 8.1 enstead of 9.0 through RaspAnd Oreo to see if an earlier version of Android would run properly with the touchscreen. The RPi succesfully booted into Android 8.1 and was able to send display signals to the touchscreen. However, due to both limitations with the RPi and issues with thermal throttling, Android ended up running extremely poorly and was unusuable. Both Ubuntu MATE and Mint (Debian) were then used as alternatives. Unfourtunately, Mint was unable to boot on the RPi (even after I tried earlier releases and compilations for other ARM instruction sets). Ubuntu MATE did manage to boot on RPi but a bug relating to account password and the onscreen keyboard also rendered it unusable.
